    <h3>Free webinar "High-performance Oracle 11g: SQL query optimization"</h3>

    <br/>
    Igor Mironchik
    <br/>
    22 May 2013
    <br/><br/>

    Oracle DBMS 11g is a powerful tool for working with huge arrays of data. In order for queries to be executed correctly and on time, it is necessary to learn how to optimize them. In this webinar, Igor Mironchik — an independent Oracle technology expert — will talk about the most effective ways to optimize SQL queries, including the use of indexes and partitioned tables, managing the Automatic Workload Repository (AWR) and SQL Plan Management (SPM).

    <br/><br/>

    <strong>Webinar program:</strong>

    <br/><br/>

    <ul>
    	<li>General Oracle Server architecture from an optimization perspective: SGA, PGA, server configuration parameters</li>
    	<li>SQL query monitoring, execution plan optimization: SQL Plan Management mechanism and AWR repository</li>
    	<li>Optimization of data storage structures to improve SQL query performance</li>
    	<li>The need to use indexes and materialized views</li>
    </ul>

    <h3>Free webinar "PL/SQL: all basic commands in one webinar"</h3>

    <br/>
    Igor Mironchik
    <br/>
    29 April 2013, 19:00
    <br/><br/>
    Oracle technologies are a huge world in which it is difficult for an IT specialist to figure it out on their own. Even if you already know SQL, to start working with Oracle, you need to master specific things within the special programming language PL/SQL. In just one webinar, Igor Mironchik — a legendary instructor and one of the veterans of Oracle training — will teach you all the basic PL/SQL commands using a live example!

    <br/><br/>

    <strong>Webinar program:</strong>

    <br/><br/>

    <ul>
    	<li>General Oracle Database Server architecture</li>
    	<li>Interaction with the server from the client side</li>
    	<li>SQL and PL/SQL: one task, two languages – working together?</li>
    	<li>Basic PL/SQL commands</li>
    	<li>Live demo: working with SQL console</li>
    	<li>Q&A session</li>
